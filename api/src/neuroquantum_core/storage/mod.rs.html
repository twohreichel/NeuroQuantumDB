1//! `NeuroQuantumDB` Storage Engine
2//!
3//! Provides persistent file-based storage with DNA compression, B+ tree indexes,
4//! and ACID transaction support for production deployment.
5//!
6//! # Architecture
7//!
8//! The storage engine is organized into several submodules:
9//!
10//! - [`engine`]: Core storage engine implementation
11//! - [`types`]: Core type definitions (Value, `DataType`, `TableSchema`, etc.)
12//! - [`row`]: Row representation and compressed storage format
13//! - [`query`]: Query types for SELECT, INSERT, UPDATE, DELETE
14//! - [`id_generation`]: ID generation strategies (`AutoIncrement`, UUID, Snowflake)
15//! - [`transaction_log`]: Transaction and operation logging
16//! - [`stats`]: Query execution statistics and database metadata
17//! - [`encryption`]: Data-at-rest encryption
18//! - [`backup`]: Backup and restore functionality
19//! - [`btree`]: B+ tree index implementation
20//! - [`buffer`]: Buffer pool management
21//! - [`pager`]: Page-based storage management
22//! - [`wal`]: Write-ahead logging
23//! - [`migration`]: Schema migration support
24//!
25//! # Example
26//!
27//! ```ignore
28//! use neuroquantum_core::storage::{StorageEngine, TableSchema, Row, Value, DataType};
29//!
30//! # async fn example() -> anyhow::Result<()> {
31//! // Create storage engine
32//! let mut storage = StorageEngine::new("./data").await?;
33//!
34//! // Create a table
35//! let schema = TableSchema::new("users", "id", vec![
36//!     ColumnDefinition::new("id", DataType::BigSerial),
37//!     ColumnDefinition::new("name", DataType::Text),
38//! ]);
39//! storage.create_table(schema).await?;
40//!
41//! // Insert a row
42//! let mut row = Row::new();
43//! row.set("name", Value::text("Alice"));
44//! let id = storage.insert_row("users", row).await?;
45//! # Ok(())
46//! # }
47//! ```
