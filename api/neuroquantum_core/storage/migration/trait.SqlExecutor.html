Expand description

Trait for executing SQL statements during migrations.

This trait allows the migration executor to execute SQL statements against the database without depending on the query engine crate directly. Implementations of this trait can be injected from higher-level crates like neuroquantum-api that have access to the full query engine.

§Example

use std::sync::Arc;
use neuroquantum_core::storage::migration::{SqlExecutor, SqlExecutionResult};

struct MyQSQLExecutor;

#[async_trait::async_trait]
impl SqlExecutor for MyQSQLExecutor {
    async fn execute_sql(&self, sql: &str) -> anyhow::Result<SqlExecutionResult> {
        // Execute the SQL statement
        Ok(SqlExecutionResult {
            rows_affected: 0,
            message: None,
        })
    }
}
